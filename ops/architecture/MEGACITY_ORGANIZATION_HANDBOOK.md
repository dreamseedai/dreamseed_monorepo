# 📌 0. 소개 (Introduction)

DreamSeedAI Organization Handbook은 DreamSeedAI 팀 전체가 공유하는 **문화·원칙·운영 기준**을 정의한 공식 문서입니다.

이 문서는 엔지니어링·AI·콘텐츠·PM·운영·디자인 등 DreamSeedAI MegaCity를 구성하는 모든 팀의 공통 규범으로 사용됩니다.

본 핸드북은 DreamSeedAI의 철학인:

```
"교육은 도시이고, AI는 그 도시의 전기(전력)다."
```

를 중심으로 작성되었습니다.

# 🧭 2. Core Principles (핵심 원칙 10가지)

DreamSeedAI는 다음 10가지 원칙 위에서 움직입니다.

1. **User First** — 학생·학부모·교사가 중심이다.
2. **Bias-free AI** — AI는 편향 없이 공정해야 한다.
3. **Educate with Empathy** — 배려 있는 교육.
4. **Move Fast, Don't Break Trust** — 신뢰를 깨지 않는 속도.
5. **Documentation First** — 문서가 곧 제품의 일부.
6. **Security by Default** — 보안은 기본값.
7. **Data Minimization** — 최소한의 정보만 수집.
8. **Privacy-Respecting AI** — 얼굴/음성은 항상 보호.
9. **Global-first Design** — 한국·미국·아시아 학생 모두 사용 가능하게.
10. **Sustainable Execution** — 무리하지 않는 지속 가능한 성장.

# 🤝 3. Team Culture (DreamSeedAI 팀 문화)

## 3.1 원칙

* 책임은 개인이 아닌 **역할(Role)** 에 있다.
* 잘못을 두려워하지 않고, 숨기지 않는다.
* 빠른 커뮤니케이션, 빠른 실험, 빠른 문서화.
* 서로를 "엔지니어/교사/PM"으로 보지 않고 **동료(Partner)** 로 본다.

## 3.2 회의 문화

* 회의는 최대 30분.
* 아젠다 없는 회의 금지.
* 회의 결론은 항상 문서로 남긴다.
* 결정된 내용은 그 자리에서 TODO로 변환.

## 3.3 실수/문제 문화

```
Blame(비난)이 아니라, Fact(사실) + Fix(해결) + Future(재발 방지)만 다룬다.
```

# 📣 4. Leadership Principles (리더십 원칙)

DreamSeedAI는 아래 7개의 리더십 원칙을 공유합니다.

1. **Long-term Thinking** — 단기성과보다 지속성 중요.
2. **High Standards** — 품질 기준을 타협하지 않는다.
3. **Bias for Action** — 결정이 늦으면 성장도 늦다.
4. **Dive Deep** — 문제를 끝까지 파본다.
5. **Act as an Owner** — 회사/도시 전체 관점에서 판단한다.
6. **Think Global** — 지역 서비스가 아니라 글로벌 플랫폼.
7. **Lead with Clarity** — 명확한 목표·문서·책임.

# 🧑‍💻 5. Communication & Collaboration

## 5.1 기본 규칙

* Slack은 공개 채널 우선
* DM은 최소화
* 중요한 논의는 반드시 문서화

## 5.2 언어 원칙

* 코드는 영어
* 문서는 한국어·영어 혼용 가능
* 국제 협업은 영어 기본

## 5.3 정보 공유 원칙

* 숨기지 않는다
* 불완전해도 공유한다
* 문서가 오래된 경우 "OUTDATED" 택 표시

# 🛠️ 6. Product Development Process

## 6.1 기본 사이클

```
Idea → Design → Document → Build → Test → Release → Review
```

## 6.2 요구사항 정의

* PRD(Product Requirement Doc) 작성 필수
* Acceptance Criteria 명확하게 작성

## 6.3 스프린트 원칙

* 2주 단위
* Sprint Goal 1~2개만 설정
* Demo 필수

## 6.4 품질 원칙

* 100% 문서화
* 자동화 테스트 필수
* 기능 추가보다 안정성 우선

---

# 🧬 7. Engineering Standards

## 7.1 Code Quality

* PEP8/Black/Flake8 준수
* Reusable Component 우선

## 7.2 API 규칙

* REST + OpenAPI 스펙 준수
* 모든 API는 SLO(p95 < 300ms) 목표

## 7.3 테스트 원칙

* Unit test 70% 이상
* DB 테스트는 transaction rollback 방식

## 7.4 보안 개발 규칙

* 인증 없는 API 금지
* SQL Injection 예방
* 출력 인코딩 기본
